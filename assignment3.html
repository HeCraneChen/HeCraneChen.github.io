1. Starting on the theoretical part (35%)

Classify all the artwork in this link https://mcescher.com/gallery/symmetry/ into 18 baskets. This include 17 symmetry groups, and none of the 17 groups.

Expected submission:

An html report that includes a big table. The table should have 18 rows, two columns. Column 1 says which symmetry group, column2 includes thumbnails of the artwork that got classified in that group.

2. Starting on the software part (92% tops)

The goal is to write a graphical user interface that can tile the entire plane with one shape. The size of "the entire plane" is defined by the zooming-in and zooming-out of the mouse.

Step1: Download data

Data is provided in HW3_data.zip file on Canvas. Find the image that pops the most to you, identify it's symmetry group. (Important note: We are not allowed to distribute the data because they are protected with copyright, owned by M.C. Escher Foundation. If you want to post your software, you need to use data created by yourself.)

Step2: Use Adobe Illustrator to trace your favorate data from provided the zip. (5%)

Northeastern provides you access to Creative Cloud. Use your NU email and password to log in.

Step3: Implement the tessellation logic. (15% per symmetry group, 3 tops)

Write the code to tile the entire plane by repeating one piece of your traced tile. Hint: This would be an iterative algorithm that rotate/translate/mirro the basic tile that you traced with Illustrator. If you are doing 3, then you need to trace 3, they need to come from different symmetry groups. Same symmetry group counts as one.

Step4: Tile the sphere (20%)

You will need to grab the tile data by logging data this Matlab code that we ran during the lecture. This would not be one of the 17 symemtry groups of Euclidean plane, but one of the 14 symmetry groups on the sphere.

Step5: Write the graphical user interface. (35%)

Write the visualization logic. Major feature is adding the adaptive the mouse control. When you zoom out, a larger area of the window will be visible, that means you need to run more iterations. Programming language of the GUI can be flexible. It depends on your goals. I recommend C++ with ImGUI (https://github.com/ocornut/imgui). But this doesn't mean Matlab/Python/Javascript/Swift has to be ruled out. It's up to what kind of application you find cool.

Extra Credit: 1 GUI Contest Winner (7%)

This is HW a bit special. We are using the art created by artist. So it's more about the GUI. Kaleido Tile that we played with during the lecture should be considered as role model.

Expected submission:

  • Data: A folder of all the the data you used, this include both the .ai file from Illustrator, and the .png file of the image.
  • Source code: A folder of all the source code and data you used.
  • All data involved: Include the example images for unit tests with provided data and any custom data you used for your very own ray tracing process.
  • HTML file with a written summary of what you did and visually showcasing your GUI
