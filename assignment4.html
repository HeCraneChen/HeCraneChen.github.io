How does that relate to the paper/bookchapters.

Well-commented source code.

2. Starting on your creative demo.

The goal is to create a deformation in Houdini, using the Dynamic Kelvinlets plugin. The deformatioin should aim to be as fun and creative as you can imagine.

Step1: Imagine.

What would be a cool animation you want to show others? From the in-class coding practice, we've seen other people come us with dancing elephant, deforming sponge, using rubber band to deform a duck ballon, moving the arms of an octupus, making the armadillo man dance. What else might be cool? What's your story?

Step2: Find a data you are interested in working with.

Could be any obj file of a 3D model that aligns with your story.

Step3: Implement it in Houdini.

You should start with double clicking the .hipnc file comes with Dynamic Kelvinlets. It loads a deformable square. Try to replace the unit test square with your story. Recall that in the lecture, we discussed translation, pinch, scale, twist. The force can be either an impulse or a push.

Expected submission:

  • Data: A folder of all the the data you used.
  • Source code: A folder of the Houdini scene .hipnc file and all the source code involved.
  • HTML file with a written summary of what you did and visually showcasing your art
